# grammar/ — vocabulary authoring discipline

*Two files. The 11-layer grammar stack + the bootstrap procedure. Read both during a project's Vocabulary Session (hard rule 12); refer back when the vocabulary needs to evolve.*

---

## Reading order

1. **`PRINCIPLES.md`** — the 11-layer grammar stack + non-negotiable commitments + the eight supervised-evolution proposal types. Foundational; the rest of the kit assumes you've read this.
2. **`BOOTSTRAP.md`** — the Vocabulary Session procedure. A 12-step process the Architect + Agent run together at project start to produce `signals/0.1.json`.

---

## What's deliberately not here

Earlier kit revisions included three additional files: `EMPIRICAL.md` (how each original project authored its grammar), `PER_LAYER.md` (per-layer detailed guidance for all 11 layers), `EXAMPLES.md` (10 worked vocabulary examples). All useful; none load-bearing for a Tier-1 LLM agent + a human Architect running the loop.

If a project needs that material, it remains available in the original `sdd-kit/grammar/` folder (the audit-trail-preserving predecessor of this kit). Reference it directly when relevant; do not copy it forward — the simplest kit lives or dies on staying small.
