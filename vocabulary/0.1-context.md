# cascade-img vocabulary v0.1 — how to read the event stream

*A companion to `vocabulary/0.1.json`. The JSON is the source of truth; [`0.1-reference.md`](0.1-reference.md) is the generated per-tag reference (payload fields, emitters, allowed values); this doc explains how the events fit together.*

## Why a fixed vocabulary

This vocabulary names every event cascade-img emits at runtime — 48 tags across 11 categories, covering the bridge daemon, the MCP server, the CLI, the prompt composer, and the curation primitives. Because every runtime action maps to a named event with a typed payload, a reader or an agent can reconstruct what the program did from the event stream alone, instead of parsing free-text prose logs.

In scope: the v0.1 surface (Midjourney Discord backend + composer + curation + log + MCP + CLI). Future backends (Flux, DALL-E, Imagen) bring their own backend-specific tags at v0.3+ — e.g. `FLUX_FIRED` / `FLUX_RECEIVED` / `FLUX_FAILED` when the Flux-via-Fal backend lands.

## The four entities

Every `job_id` or `asset_id` in a payload points at one of four durable things (declared under `ontology.entities` in the JSON):

- **Job** — one /imagine roll through to its terminal state (done | failed). Identified by `job_id`.
- **Asset** — a named output the user is generating (e.g. `"mountain-icon"`, `"hero-portrait"`). Identified by `asset_id`.
- **Bridge** — the long-running Flask + discord.py-self daemon process, operated through HTTP.
- **Backend** — the pluggable backend implementing `ImageGenerationBackend`. v0.1 has one (`MidjourneyDiscordBackend`); the `backend` field on `CASCADE_INIT` says which is active.

## The happy path

A normal generation run produces this sequence:

```
CONFIG_VALIDATED → CASCADE_INIT → DISCORD_CONNECTED
  → IMAGINE_FIRED → GRID_MATCHED → GRID_RECEIVED
  → (UPSCALE_REQUESTED → UPSCALE_RECEIVED)*   # zero, one, or four pairs
  → JOB_COMPLETED
```

Daemon shutdown ends the stream with `BRIDGE_SHUTDOWN`. An MCP session is framed the same way: `MCP_SERVER_STARTED` opens it, each tool call fires `MCP_TOOL_CALLED` then exactly one of `MCP_TOOL_COMPLETED` / `MCP_TOOL_FAILED`, and `MCP_SERVER_STOPPED` closes it. A CLI roll wraps the job sequence in `CLI_ROLL_STARTED` / `CLI_ROLL_COMPLETED` (or `CLI_ROLL_FAILED`).

Expected timing for `IMAGINE_FIRED → JOB_COMPLETED` (declared in `temporal_invariants`): 180s when `upscale` is **none** (grid-only), 360s when **single** (a `1`–`4` upscale), 600s when **all**. Each window's `applies_when.upscale` selects the band by matching the run's `IMAGINE_FIRED.payload.upscale` (null/absent → none, `1`–`4` → single, `all` → all), so a run is judged against the band for its own mode. A run past its band carries `severity: "warning"` — it is flagged as stalled, not failed: a slow MJ day is a warning, never a gate failure.

## When things go wrong

Tags marked `incident` in the JSON report failures. The two with stable, enum-locked codes:

- **`JOB_FAILED.error_code`** — `DISCORD_400_OUTDATED`, `DISCORD_400_UNKNOWN_CHANNEL`, `DISCORD_401`, `DISCORD_HTTP_ERROR`, `DISCORD_NOT_READY`, `GRID_DOWNLOAD_FAILED`, `MJ_UUID_MISSING`, `SUBMIT_FAILED`, `UPSCALE_DOWNLOAD_FAILED`, `UPSCALE_BUTTON_FAILED`, `UPSCALE_ALL_BUTTONS_FAILED`, `RESUBMIT_REQUIRED`. The same code appears in the `GET /wait` HTTP response, so a client can map it to remediation.
- **`CONFIG_VALIDATION_FAILED.code`** — `MISSING_DISCORD_TOKEN`, `MISSING_CHANNEL_ID`, `INVALID_CHANNEL_ID`, `MISSING_IMAGINE_VERSION`, `INVALID_PORT`. This one can end the process before `CONFIG_VALIDATED` ever fires.

Partial-failure incidents (`UPSCALE_PRESS_FAILED`, `UPSCALE_DOWNLOAD_DROPPED`, `JOB_SUBMIT_TIMEOUT`, `MJ_DERIVED_FAILED`, `OUTPUT_PATH_COLLISION`) do **not** terminate the job — see each tag's note in the reference for what survives and what the client should do.

## Tag-to-artifact table

cascade-img has no UI; its observable outputs are file outputs (PNGs written to disk) and HTTP responses (structured JSON returned to clients). This table pairs each behavior tag with the artifact it corresponds to:

| Behavior tag | Artifact counterpart | Notes |
|---|---|---|
| `IMAGINE_FIRED` | HTTP response: `{job_id, asset_id, status: "submitted", upscale}` from `POST /imagine` | Tag fires after the response is returned. |
| `GRID_MATCHED` | None at the consumer surface — internal state transition | Visible only in the event stream. |
| `GRID_RECEIVED` | PNG file at `{output_dir}/{asset_id}.{png|webp}` | The `path` and `bytes` payload fields identify the artifact. |
| `UPSCALE_RECEIVED` | PNG file at `{output_dir}/{asset_id}_u{slot}.{png|webp}` or `.png` | Same shape. |
| `JOB_COMPLETED` | HTTP response: `{status: "done", image_path, grid_path, upscale_paths}` from `GET /wait` | Tag fires before the response is returned. |
| `JOB_FAILED` | HTTP response: `{status: "failed", error, error_code}` from `GET /wait` | Stable `error_code` mapped to remediation. |
| `MJ_DERIVED_RECEIVED` | PNG/WebP at `{output_dir}/{asset_id}_{kind}_{uuid8}.{ext}` | A vary/zoom/pan/upscale/animation result from an `mj_action` press. |
| `ASSET_PROMOTED` | PNG file at consumer's `dest` path | Idempotent — overwrites. |
| `PROMPT_LOGGED` | JSONL line in `{prompt_log_path}` | Append-only. |

## Design rules

The JSON encodes more than a tag list; these are the rules it enforces or declares:

- **Naming** — tags are `{ENTITY|SUBSYSTEM}_{VERB_PAST_PARTICIPLE}`: `RECEIVED` not `RECEIVE`, `VALIDATED` not `VALIDATE`, `COMPLETED` not `COMPLETE`. The same event always carries the same identifier.
- **Strict at emit** — `Vocabulary.validate()` raises on an unknown tag, a missing required field, an undeclared field, or a value outside an enum-locked field (the `evidence_constraints` block — e.g. `GRID_MATCHED.match_path` must be `pending` / `progress_fallback` / `unknown`). `CASCADE_STRICT_SIGNALS=false` relaxes the check in production.
- **Sessions** — `sessions.strata` declares two framing pairs: `CASCADE_INIT` … `BRIDGE_SHUTDOWN` (the daemon, carrying `package_version` + `backend`) and `MCP_SERVER_STARTED` … `MCP_SERVER_STOPPED` (carrying `transport`: `stdio` | `http`).
- **Ordering** — `state_transitions.rules` declares the must-precede graph: `CONFIG_VALIDATED → CASCADE_INIT` (the daemon validates config before it announces init; `--check-env`/`--doctor` validate without starting); `DISCORD_CONNECTED` before any `IMAGINE_FIRED`; `GRID_MATCHED → GRID_RECEIVED` per `job_id`; `UPSCALE_REQUESTED → UPSCALE_RECEIVED` per `job_id`+`slot`. `JOB_COMPLETED` / `JOB_FAILED` are terminal — no later job-**lifecycle** tag (the `forbidden_tags_after` set: `IMAGINE_FIRED`, `GRID_*`, `UPSCALE_*`, the two terminals themselves) carries the same `job_id`. The terminal rule's `exempt_tags` are the post-terminal events that legitimately do carry a completed job's `job_id`: `MJ_ACTION_REQUESTED` / `MJ_ACTION_FAILED` and `MJ_DERIVED_RECEIVED` / `MJ_DERIVED_FAILED` (an `mj_action` press and its derived result act on an already-DONE job), `JOB_EVICTED` (only DONE/FAILED jobs are evicted), and `OUTPUT_PATH_COLLISION` (can fire on a derived download). These are additive by design — the earlier "no further job tag" phrasing was false for the system as built.
- **Known emitters** — `operators.operators` maps every tag to the component and source file that emits it, so any event traces back to code.

## Versioning: what "locked" means

`0.1.json` says `locked: true` (2026-06-02). Locked means the **existing** tags are frozen: no renaming, no removal, no payload change to a tag that shipped. The vocabulary still **grows** within 0.1 — new tags may be added (declaration-before-emit, enforced by the parity check), and notes or source locations may be corrected. Anything that would break a consumer reading old streams — a rename, a removal, a payload change — bumps to 0.2, where removals become deprecation entries rather than deletes.

## Adding a tag

1. Add the entry to `vocabulary/0.1.json` **and** the package-data copy at `packages/python/src/cascade_img/vocabulary/versions/0.1.json` (kept identical; CI reads the package copy).
2. Regenerate the reference: `python3 packages/python/tools/render_vocabulary_reference.py`.
3. Then write the `emit()` callsite. CI's parity check (`packages/python/tools/check_vocabulary_parity.py`) fails on any emit of an undeclared tag, and the reference check fails if `0.1-reference.md` is stale.
