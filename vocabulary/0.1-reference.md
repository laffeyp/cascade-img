A response-message action could not be pressed. error_code is NO_UPSCALED_IMAGE (the job has no upscaled image to act on), BUTTON_NOT_FOUND (no matching button on the live message), DISCORD_NOT_READY, or HTTP_<status>.

- Payload: `asset_id`, `job_id`, `parent_message_id`, `action_kind`, `mj_uuid`, `path`, `bytes`, `content_type`
- Emitted by `BridgeDaemon` (cascade_img/backends/midjourney_discord/bridge.py)
- `action_kind` must be one of: `variation`, `zoom`, `pan`, `upscale`, `animation`

A derived result produced by an mj_action press landed and was downloaded to the parent asset. Routed by Discord message_reference == the SOLO upscaled-image message id (upscale_message_id) — the only signal present on every family; recency is unsafe because the MJ channel is shared. action_kind is the family classified from MJ's content suffix (variation/zoom/pan/upscale/animation). animation lands as an ANIMATED WEBP (content_type image/webp, NOT video/mp4). mj_uuid is the NEW MJ job uuid minted for the derived result. favorite presses produce no artifact and emit nothing here.

- Payload: `asset_id`, `job_id`, `parent_message_id`, `action_kind`, `error_code`, `error_message`
- Emitted by `BridgeDaemon` (cascade_img/backends/midjourney_discord/bridge.py)
- `action_kind` must be one of: `variation`, `zoom`, `pan`, `upscale`, `animation`

A derived result was matched to its parent job but could not be downloaded. The parent job stays DONE — a derived result is additive, so a download failure is an incident on the derived artifact, not a job failure. error_code is the exception class or HTTP status.

### `MJ_ACTION_SURFACE_REGISTERED`

- Payload: `asset_id`, `job_id`, `slot`, `message_id`, `surface_kind`
- Emitted by `BridgeDaemon` (cascade_img/backends/midjourney_discord/bridge.py)
- `surface_kind` must be one of: `image_upscale`, `video_solo`

A downloaded result became a pressable action surface: the per-slot SOLO message a follow-on mj_action targets (recorded in upscale_message_ids[slot]). surface_kind is image_upscale (a U1-U4 SOLO — source of vary/zoom/pan/animate/favorite/re-upscale) or video_solo (a video_virtual_upscale SOLO — source of extend_high/extend_low; its extend buttons are grid-aligned, so slot N round-trips). Emitted just before the same artifact's UPSCALE_RECEIVED or MJ_DERIVED_RECEIVED so the surface is observable the instant the result is — closing the action chain's silent middle link: MJ_ACTION_REQUESTED → (download) → MJ_ACTION_SURFACE_REGISTERED → the next MJ_ACTION_REQUESTED targets this slot.

## curation

### `CROP_QUADRANT`

- Payload: `quadrant`, `w`, `h`
- Emitted by `CurationKit` (cascade_img/curation/)

crop_quadrant() emitted a cropped image. quadrant=0 means the whole image was returned (single-upscale passthrough), 1-4 means a grid quadrant. w/h are the OUTPUT dimensions.

### `ALPHA_KEY_APPLIED`

- Payload: `tolerance`, `bg_r`, `bg_g`, `bg_b`, `keyed_count`, `total_count`
- Emitted by `CurationKit` (cascade_img/curation/)

alpha_key_corners() processed an image. bg_r/bg_g/bg_b are the four-corner average. keyed_count/total_count gives the proportion of pixels keyed transparent — typical sprite values are 30-70%. A keyed ratio outside that band suggests tolerance is wrong (too tight = halo, too loose = sprite eaten).

### `ASSET_PROMOTED`

- Payload: `src`, `dest`, `bytes`
- Emitted by `CurationKit` (cascade_img/curation/)

promote() copied a curated asset to the consumer's asset tree.

### `CONTACT_SHEET_RENDERED`

- Payload: `src`, `dest`, `tiles`, `w`, `h`
- Emitted by `CurationKit` (cascade_img/curation/)

contact_sheet() composited a grid's panels (with optional index/score labels) into one annotated image — a far better single input for vision-model selection than four separate reads. tiles is the number of panels placed; w/h are the output sheet dimensions.

### `AUTO_TRIM_APPLIED`

- Payload: `src`, `dest`, `bbox`, `w`, `h`
- Emitted by `CurationKit` (cascade_img/curation/)

auto_trim() cropped an image to its content bounding box. bbox is [left, top, right, bottom] in source pixels; w/h are the trimmed output dimensions. A bbox spanning the whole image means nothing was trimmed.

### `VIDEO_FILMSTRIP_RENDERED`

- Payload: `src`, `dest`, `frame_count`, `duration_s`
- Emitted by `?` (?)

video_filmstrip() sampled N keyframes of an animated video (webp) into one vision-readable contact sheet so an agent can inspect a temporal artifact the way it reads a still (F32). frame_count/w/h are the source video signature (duration_s/fps are best-effort — 0 when the source carries no per-frame duration, as PIL reports for many webp); dest is the filmstrip PNG.

### `LOOP_SEAM_MEASURED`

- Payload: `src`, `frame_count`, `delta`
- Emitted by `?` (?)

loop_seam_delta() measured how cleanly a looping video closes: delta is the normalized (0-1) pixel distance between the last and first frames (low = seamless loop), reported as a number instead of a human eyeballing the seam (F33). The video analog of the audio click-at-the-loop-point check.

### `PALETTE_QUANTIZED`

- Payload: `src`, `dest`, `n_colors`, `method`
- Emitted by `CurationKit` (cascade_img/curation/)

palette_quantize() reduced an image to a fixed palette for the limited-palette look. method is 'median_cut', 'maximum_coverage', or 'octree'; n_colors is the requested palette size.

### `SPRITE_SHEET_PACKED`

- Payload: `dest`, `count`, `layout`, `w`, `h`
- Emitted by `CurationKit` (cascade_img/curation/)

sprite_sheet() packed N curated sprites into one atlas plus a JSON frame map written alongside it. layout is 'grid', 'row', or 'column'; count is the number of frames; w/h are the atlas dimensions.

### `GRID_SCORED`

- Payload: `src`, `tiles`, `top_slot`
- Emitted by `CurationKit` (cascade_img/curation/)

score_grid() ranked the four quadrants of a 2x2 grid on sharpness/contrast/edge-density so an agent picks on evidence instead of a blind quadrant int. tiles is the number scored (4); top_slot (1-4) is the highest composite score. Advisory — the agent still confirms with vision.
